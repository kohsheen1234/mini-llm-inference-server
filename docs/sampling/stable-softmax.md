# Numerically stable softmax

> Step 1 of the [roadmap](../../README.md#roadmap). Implemented in
> [`mini_llm_server/sampling.py`](../../mini_llm_server/sampling.py) as `stable_softmax`.

The first component of the pipeline converts model outputs called **logits**
into probabilities.

## What are logits?

A model produces one score for every token in the vocabulary. For example:

```python
logits = np.array([2.0, 1.0, 0.1])
```

These values are not probabilities. They:

- can be negative,
- can be larger than 1,
- do not need to sum to 1.

Softmax converts them into a probability distribution:

```text
logits       → softmax → probabilities
[2, 1, 0.1]             [0.659, 0.242, 0.099]
```

The output values are non-negative and sum to 1.

## Basic softmax formula

Softmax turns a vector of logits $x = (x_1, x_2, \dots, x_n)$ into a probability
distribution. For each entry $x_i$:

$$\text{softmax}(x)_i = \frac{e^{x_i}}{\sum_{j=1}^{n} e^{x_j}}$$

Three properties follow directly from this definition, and together they are
exactly what "a probability distribution" means:

- **Non-negative.** $e^{x_i} > 0$ for every real $x_i$, so every output is positive.
- **Sums to one.** The denominator is the sum of the numerators, so
  $\sum_{i=1}^{n} \text{softmax}(x)_i = 1$.
- **Order-preserving.** $e^{x}$ is strictly increasing, so a larger logit always
  maps to a larger probability. Softmax reshapes the scores without reordering them.

A direct implementation would be:

```python
exp_values = np.exp(logits)
probabilities = exp_values / np.sum(exp_values)
```

This works for small values, but it can fail for large logits.

## Why the naive version is unsafe

Floating-point numbers have a finite range. The largest finite value a 64-bit
float can hold is about $1.8 \times 10^{308}$, and

$$e^{x} > 1.8 \times 10^{308} \quad\text{once}\quad x > \ln(1.8 \times 10^{308}) \approx 709.8$$

So any logit above roughly **710** overflows `np.exp` to `inf` (panel A above).
Concretely, consider:

```python
logits = np.array([1000.0, 999.0, 998.0])
np.exp(1000.0)   # too large for float64
```

The result becomes `inf`, and the later division `inf / inf` produces `NaN`.
That `NaN` propagates into token sampling and breaks generation. Note the logits
themselves are perfectly ordinary — it is only the intermediate `exp` that
overflows.

## The stability trick

Softmax is **invariant to shifting all logits by the same constant**. For any
constant $c$:

$$\text{softmax}(x - c)_i = \frac{e^{x_i - c}}{\sum_{j} e^{x_j - c}}
= \frac{e^{x_i}\,e^{-c}}{\sum_{j} e^{x_j}\,e^{-c}}
= \frac{e^{-c}}{e^{-c}} \cdot \frac{e^{x_i}}{\sum_{j} e^{x_j}}
= \text{softmax}(x)_i$$

The shared factor $e^{-c}$ cancels top and bottom, so the output is identical
(panel B above). We are free to pick any $c$ — so pick the one that makes the
exponentials safe:

$$c = \max_{j} x_j$$

With this choice every shifted logit satisfies $x_i - c \le 0$, so
$e^{x_i - c} \in (0, 1]$:

- **No overflow:** the largest exponent is $e^{0} = 1$, well within range.
- **No divide-by-zero:** at least one term equals $1$, so the denominator is $\ge 1$.

For `[1000, 999, 998]`, subtracting the maximum gives `[0, -1, -2]`, and
exponentiation is now safe: `[e^0, e^-1, e^-2] = [1.0, 0.3679, 0.1353]`. The
relative differences — and therefore the final probabilities — are unchanged.

## The log-sum-exp view

The same trick has a name outside softmax. The denominator's logarithm is the
**log-sum-exp** (LSE) function:

$$\text{LSE}(x) = \log \sum_{j} e^{x_j}$$

and softmax in log-space is simply $\log \text{softmax}(x)_i = x_i - \text{LSE}(x)$.
Computed naively, `LSE` overflows for the same reason softmax does. The stable
form factors the max back out:

$$\text{LSE}(x) = \max_{j} x_j + \log \sum_{j} e^{x_j - \max_{k} x_k}$$

This is the identical "subtract the max, then exponentiate" pattern, and it is
why the implementation shifts by `np.max` before calling `np.exp`.

## The implementation, line by line

```python
def stable_softmax(logits):
    maximum = np.max(logits, axis=-1, keepdims=True)
    shifted = logits - maximum
    exponentials = np.exp(shifted)
    total = np.sum(exponentials, axis=-1, keepdims=True)
    return exponentials / total
```

**`axis=-1`** applies the operation along the final dimension. This matters
because logits may arrive in several shapes — `(vocab,)` for one request,
`(batch, vocab)` for a batch, `(batch, seq, vocab)` for a sequence batch — and in
every case the final axis holds the vocabulary scores. For a `(2, 3)` input,
`np.max(logits, axis=-1)` returns one maximum per row.

**`keepdims=True`** preserves the reduced dimension: the max of a `(2, 3)` array
becomes `(2, 1)` instead of `(2,)`. That preserved dimension lets NumPy broadcast
each row's maximum across its row when computing `logits - maximum`.

**`shifted = logits - maximum`** makes the largest value in each row zero and
every other value zero or negative, which is what keeps `np.exp` from
overflowing.

**`total` and the final division** sum the exponentials along the vocabulary axis
(again with `keepdims=True` for broadcasting) and normalise, so the result sums
to 1 along the last axis.

## Examples

```python
import numpy as np
from mini_llm_server import stable_softmax

logits = np.array([2.0, 1.0, 0.1])
probs = stable_softmax(logits)
print(probs)          # [0.65900114 0.24243297 0.09856589]
print(np.sum(probs))  # 1.0
```

Batched input — each row is a separate distribution, and large logits stay
stable:

```python
logits = np.array([
    [2.0, 1.0, 0.0],
    [1000.0, 999.0, 998.0],
])
probs = stable_softmax(logits)
print(np.sum(probs, axis=-1))  # [1.0, 1.0]
```

## Where softmax sits in inference

During generation the model produces one logit per vocabulary token; softmax
turns those scores into probabilities the sampler can draw from:

```text
transformer output → logits → temperature and filtering → stable softmax → probabilities → token sampling
```

## Correctness requirements

The implementation must satisfy:

- **Same shape** — output shape matches input `(..., vocab)`.
- **Non-negative** — every probability is at least zero.
- **Sums to one** — `np.sum(probabilities, axis=-1)` is approximately `1.0`.
- **Numerical stability** — very large positive or negative logits must not
  produce `inf` or `NaN`.
